# Knowledge Base

Add `.md` or `.txt` files here. They are automatically chunked and indexed for RAG.

## Folder structure

```
knowledge_base/
├── templates/            # Outreach templates, broken down by use-case
│   ├── follow-ups/
│   ├── intros/
│   └── thank-yous/
├── schools/              # School-specific playbooks/resources
│   ├── emory/
│   └── georgetown/
├── guides/               # Product + success guides
│   ├── product/
│   └── user-success/
├── playbooks/            # Scenario-based guides (e.g., interviews)
│   └── interviews/
├── data/                 # Supporting data, trackers, case studies
│   ├── stats/
│   └── stories/
└── README.md
```

## Guidelines

- Keep sections concise with clear headings (H1/H2).
- Prefer markdown lists and short paragraphs.
- One topic per file is ideal.
- Place new files in the folder that best matches the content so retrieval remains focused.
